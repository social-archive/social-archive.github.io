# MDX 작성 컨벤션

이 사이트는 **MDX**를 기본 포맷으로 사용합니다. docs, blog, pages 모두 `.mdx` 확장자를 사용합니다.

## 왜 MDX인가

- **Markdown 호환**: 기존 마크다운 문법 그대로 사용 가능
- **JSX 확장**: React 컴포넌트를 문서에 직접 삽입 가능
- **재사용**: `@site/src/components/Notion` 등 공통 컴포넌트 활용

## 파일 확장자

| 위치 | 확장자 | 비고 |
|------|--------|------|
| `docs/` | `.mdx` | 문서 페이지 |
| `blog/` | `.mdx` | 블로그 포스트 |
| `src/pages/` | `.mdx` | 독립 페이지 |

`_`로 시작하는 파일(예: `_home.mdx`)은 페이지로 생성되지 않고 import 전용입니다.

## 컴포넌트 사용 예

```mdx
import { Callout, H2 } from '@site/src/components/Notion';

<Callout emoji="📂">
  **강조**하고 싶은 내용을 넣을 수 있습니다.
</Callout>
```

## 새 문서 작성 시

1. `.md` 대신 `.mdx`로 생성
2. frontmatter는 동일하게 사용
3. 필요 시 `import`로 컴포넌트 추가
